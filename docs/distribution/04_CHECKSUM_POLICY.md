# Checksum Policy

## Overview

Every MJU-DRP registry package includes a `checksum.json` file containing SHA-256 hashes for all package artifacts. This enables consumers and CI systems to verify package integrity.

## Generation

Checksums are generated by `scripts/generate-checksum.mjs` as part of the release pipeline:

```bash
node scripts/generate-checksum.mjs
```

## Checksum Format

```json
{
  "package_version": "1.0.0",
  "registry_version": "1.0",
  "generated_at": "2026-07-06T15:30:00.000Z",
  "total_artifacts": 15,
  "artifacts": [
    {
      "filename": "document-registry.json",
      "size": 85000,
      "sha256": "abcdef1234567890..."
    }
  ]
}
```

## Verification

Consumers and CI systems can verify package integrity:

```bash
# Verify a single file
sha256sum document-registry.json

# Compare with checksum.json values
node -e "
const c = require('./checksum.json');
const fs = require('fs');
const crypto = require('crypto');
c.artifacts.forEach(a => {
  const h = crypto.createHash('sha256').update(fs.readFileSync(a.filename)).digest('hex');
  console.log(a.filename, h === a.sha256 ? 'OK' : 'MISMATCH');
});
"
```

## Policy Rules

1. **All artifacts must have checksums** — Every file in the package is checksummed
2. **Checksums are generated at package build time** — Never manually edited
3. **Mismatch = invalid package** — A checksum mismatch means the package is corrupted or tampered
4. **SHA-256 only** — No other hash algorithms are used or accepted
5. **Size is included** — File size in bytes is recorded alongside the hash for additional validation
