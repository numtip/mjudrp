# MJU-DRP Knowledge Base

## Purpose

This directory captures reusable enterprise knowledge from the MJU Document Registry Platform project. It is intended to be transferable to other MJU projects, not just MJU-DRP.

## Contents

| File | Purpose |
|------|---------|
| `CERTIFIED_TOOLS.md` | Tools and technologies that passed practical verification |
| `REJECTED_TOOLS.md` | Tools evaluated and rejected with reasons |
| `CONDITIONAL_TOOLS.md` | Tools requiring specific conditions for use |
| `LESSONS_LEARNED.md` | Key lessons from the ERC process |
| `BEST_PRACTICES.md` | Best practices discovered during certification |
| `ANTI_PATTERNS.md` | Approaches that proved counterproductive |
| `PATTERNS.md` | Reusable patterns for other projects |

## How to Use

1. **New projects** — Read `CERTIFIED_TOOLS.md` and `PATTERNS.md` first to understand what tools and approaches are validated.
2. **Tool selection** — Before evaluating a tool, check `REJECTED_TOOLS.md` to avoid repeating past evaluations.
3. **Implementation** — Follow `BEST_PRACTICES.md` and avoid `ANTI_PATTERNS.md`.

## Source

This knowledge base was created during MJU-DRP Enterprise Resource Certification (ERC) Sprint v1.4 (2026-07-06). All tools were evaluated against real registry data and workflows.
